## Environment Variables

All config is injected via Vite env vars prefixed with `VITE_`. The `.env` file is auto-generated by `env_generate.py` — never edit it manually.

```bash
# UI/.env — auto-generated, do not edit

# Agent ARNs
VITE_TREND_AGENT_ARN=arn:aws:bedrock-agentcore:us-east-1:123456789:agent-runtime/...
VITE_ANOMALY_AGENT_ARN=arn:aws:bedrock-agentcore:us-east-1:123456789:agent-runtime/...

# API endpoints (all routes in one API Gateway)
VITE_API_GET_PRODUCTS=https://xxx.execute-api.us-east-1.amazonaws.com/get-products
VITE_API_GET_SALES_SUMMARY=https://xxx.execute-api.us-east-1.amazonaws.com/get-sales-summary
VITE_API_START_REPORT=https://xxx.execute-api.us-east-1.amazonaws.com/start-sales-report
VITE_API_REPORT_STATUS=https://xxx.execute-api.us-east-1.amazonaws.com/get-report-status
VITE_API_GET_ANOMALIES=https://xxx.execute-api.us-east-1.amazonaws.com/get-anomalies
VITE_API_GET_TRENDS=https://xxx.execute-api.us-east-1.amazonaws.com/get-trend-analysis

# Cognito
VITE_USE_COGNITO_DOMAIN=https://retail-analytics-dev-pool.auth.us-east-1.amazoncognito.com
VITE_USE_COGNITO_CLIENT_ID=abc123clientid
VITE_USE_COGNITO_CLIENT_SECRET=...
VITE_USE_COGNITO_SCOPE='email openid profile'
```

Access in code:
```typescript
const apiUrl = import.meta.env.VITE_API_START_REPORT;
const agentArn = import.meta.env.VITE_ANOMALY_AGENT_ARN;
```

Never use `process.env` in Vite projects — use `import.meta.env` only.

## Api Client — Axios + Interceptors

`src/api/index.ts` creates a single Axios instance shared across the entire app.

```typescript
// src/api/index.ts
import axios from "axios";

const api = axios.create({});

// ── Request interceptor — attach Bearer token from localStorage ──
api.interceptors.request.use(
  (config) => {
    const token = localStorage.getItem("token");
    if (token) {
      config.headers.Authorization = `Bearer ${token}`;
    }
    return config;
  },
  (error) => Promise.reject(error),
);

// ── Response interceptor — handle 401 globally ──
api.interceptors.response.use(
  (response) => response,
  (error) => {
    if (error.response?.status === 401) {
      // Clear all auth state
      localStorage.clear();
      sessionStorage.clear();

      // Dispatch event so any listener (e.g. toasts) can react
      window.dispatchEvent(new CustomEvent("sessionExpired", { detail: { reason: "unauthorized" } }));

      // Redirect to login after a brief delay
      setTimeout(() => { window.location.href = "/login"; }, 250);
    }
    return Promise.reject(error);
  },
);

export default api;

// Generic call wrapper — import this in all API modules
export const apiCall = async (
  method: "GET" | "POST" | "PUT" | "DELETE" | "PATCH",
  url: string,
  data?: unknown,
) => {
  const response = await api.request({ method, url, data });
  return response.data;
};
```

Domain-specific API calls are made directly from page-level hooks using the `api` Axios instance:

```typescript
// src/pages/ForeCastAnalysis/useForeCastAnalysis.ts
import api from '../../api';

// Kick off a long-running job
const startJob = () =>
  api.get(import.meta.env.VITE_API_START_FORECAST, { params: { product_name } });

// Fetch existing results for a product
const loadResults = (productTitle: string) =>
  api.get(import.meta.env.VITE_API_VIEW_FORECAST + productTitle);
```

## Cognito Auth Flow

The UI uses OAuth2 Authorization Code flow (not Amplify): redirect to Cognito hosted UI → Cognito redirects back with `?code=` → exchange code for tokens → decode JWT for user info.

### 1 — Redirect to Cognito

```typescript
// src/context/AuthContext.tsx
const signInWithSSO = () => {
  const domain     = import.meta.env.VITE_USE_COGNITO_DOMAIN;
  const clientId   = import.meta.env.VITE_USE_COGNITO_CLIENT_ID;
  const redirectUri = import.meta.env.VITE_USE_COGNITO_REDIRECT_URI;
  const scope       = import.meta.env.VITE_USE_COGNITO_SCOPE;

  const loginUrl = [
    `${domain}/oauth2/authorize`,
    `?response_type=code`,
    `&client_id=${clientId}`,
    `&redirect_uri=${encodeURIComponent(redirectUri)}`,
    `&scope=${encodeURIComponent(scope)}`,
  ].join('');

  window.location.href = loginUrl;
};
```

### 2 — Exchange Code for Tokens

`src/context/SSOService.tsx` handles the callback at `/sso-login?code=...`:

```typescript
export const exchangeCognitoToken = async (code: string) => {
  const domain       = import.meta.env.VITE_USE_COGNITO_DOMAIN;
  const clientId     = import.meta.env.VITE_USE_COGNITO_CLIENT_ID;
  const clientSecret = import.meta.env.VITE_USE_COGNITO_CLIENT_SECRET;
  const redirectUri  = import.meta.env.VITE_USE_COGNITO_REDIRECT_URI;

  // Basic auth header: base64(clientId:clientSecret)
  const basicAuth = btoa(`${clientId}:${clientSecret}`);

  const response = await fetch(`${domain}/oauth2/token`, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/x-www-form-urlencoded',
      Authorization: `Basic ${basicAuth}`,
    },
    body: new URLSearchParams({ grant_type: 'authorization_code', code, redirect_uri: redirectUri, client_id: clientId }),
  });

  const data = await response.json();

  // Fetch user info with the access token
  // userInfo endpoint requires POST in AWS Cognito
  const userInfoResp = await fetch(`${domain}/oauth2/userInfo`, {
    method: 'POST',
    headers: { Authorization: `Bearer ${data.access_token}` },
  });
  const userInfo = await userInfoResp.json();

  // Extract Cognito group membership from the JWT (not returned by userInfo endpoint)
  const decodedToken: any = jwtDecode(data.access_token);
  const groups = decodedToken?.['cognito:groups'] || [];

  return {
    user: { ...userInfo, groups },
    tokens: {
      access_token:  data.access_token,
      id_token:      data.id_token,
      refresh_token: data.refresh_token,
    },
  };
};
```

### 3 — Store Tokens

After successful exchange, store tokens in `localStorage`:

```typescript
localStorage.setItem('token',         data.access_token);
localStorage.setItem('access_token',  data.access_token);
localStorage.setItem('id_token',      data.id_token);
localStorage.setItem('refresh_token', data.refresh_token);
localStorage.setItem('auth_user',     JSON.stringify(userInfo));
```

The Axios request interceptor automatically picks up `token` from `localStorage` on every subsequent API call.

## Bedrock Agent Client

Direct agent invocation from the browser uses `@aws-sdk/client-bedrock-agentcore`:

```typescript
// src/api/bedrock-client.ts
import { BedrockAgentCoreClient, InvokeAgentRuntimeCommand } from "@aws-sdk/client-bedrock-agentcore";

const generateSessionId = (length = 34) => {
  const bytes = crypto.getRandomValues(new Uint8Array(length));
  return Array.from(bytes, (b) => "abcdefghijklmnopqrstuvwxyz"[b % 26]).join('');
};

export const invokeAgent = async (inputText: unknown) => {
  const client  = new BedrockAgentCoreClient({ region: "us-east-1" });
  const command = new InvokeAgentRuntimeCommand({
    runtimeSessionId: generateSessionId(),  // 33+ chars — each new session = new MicroVM
    agentRuntimeArn:  import.meta.env.VITE_AGENT_ARN,   // ← env var set by env_generate.py
    qualifier:        "DEFAULT",            // optional — "DEFAULT" is the active endpoint
    payload:          { prompt: inputText },
  });

  try {
    const response    = await client.send(command);
    const apiResponse = await response.response.transformToString();
    return { result: JSON.parse(apiResponse), message: "" };
  } catch {
    return { result: null, message: "Error connecting to agent. Please try again." };
  }
};
```

**Note:** The AWS SDK client requires credentials. In this architecture, the Cognito access token is passed as the authorization header — the browser session must have a valid Cognito JWT to invoke the agent runtime.

## App Provider Tree

`src/App.tsx` wraps everything in the correct provider order. Providers towards the root are available to all children.

```tsx
// src/App.tsx
import { BrowserRouter as Router }    from "react-router-dom";
import { AuthProvider }               from "./context/AuthContext";
import { ToastProvider }              from "./context/AppGlobalToast";
import { AppGlobalContextProvider }   from "./common/AppGlobalContextProvider";
import AppRoutes                      from "./routes";

function App() {
  return (
    <ToastProvider>                      {/* Flash toasts — available everywhere */}
      <AuthProvider>                     {/* Auth state + signIn/signOut */}
        <AppGlobalContextProvider>       {/* Shared agent/job state (job status, anomaly data, RCA results) */}
          <Router>
            <AppRoutes />                {/* Route table */}
          </Router>
        </AppGlobalContextProvider>
      </AuthProvider>
    </ToastProvider>
  );
}
```

## Component Patterns

**Cloudscape components** for all UI primitives — never use bare HTML for buttons, tables, or form inputs:

```tsx
import Button      from "@cloudscape-design/components/button";
import Table       from "@cloudscape-design/components/table";
import Flashbar    from "@cloudscape-design/components/flashbar";
import SpaceBetween from "@cloudscape-design/components/space-between";

// Cloudscape Flashbar for success/error toasts
const showSuccess = (message: string) =>
  setFlashItems([{ type: "success", content: message, dismissible: true }]);
```

**Polling pattern** for async job status — polling lives in `MainLayout.tsx` and uses recursive `setTimeout` (not `setInterval`). This allows the next request to wait for the previous one to complete before scheduling another:

```typescript
// src/layouts/MainLayout.tsx
const getRunningJobStatus = () => {
  api
    .get(import.meta.env.VITE_API_FORECAST_STATUS, {
      params: { job_id: appGlobalContext?.foreRunningDetails?.job_id },
    })
    .then((res: any) => {
      if (res?.data?.status === 'PROCESSING' || res?.data?.status === 'QUEUED') {
        // Schedule next poll only after this one finishes (avoids overlap)
        setTimeout(() => { getRunningJobStatus(); }, 5000);
      } else if (res?.data?.status === 'COMPLETED') {
        handleExecutedForecastResults(res?.data?.result, true);
        updateForecastRunningStatus('');   // clears job_id → stops polling
      } else {
        handleExecutedForecastResults([], false);
        updateForecastRunningStatus('');
      }
    });
};

// Triggered when job ID is set in global context:
useEffect(() => {
  if (appGlobalContext?.foreRunningDetails?.runningStatus) {
    getRunningJobStatus();
  }
}, [appGlobalContext?.foreRunningDetails?.runningStatus]);
```

**Why in `MainLayout`?** Polling must survive page navigation. Because the layout wraps all pages, it stays mounted even when the user switches pages — so the notification arrives regardless of which page they're on when the job finishes.

## TypeScript Interface Conventions

Two interface files exist:

**`src/interfaces/index.ts`** — shared auth/API shapes:

```typescript
export interface User {
  id: string;
  email: string;
  name: string;
}

export interface ApiResponse<T> {
  success: boolean;
  data?: T;
  message?: string;
}
```

**`src/interfaces/useInterfacesAndInitialValues.ts`** — page-level form and submission shapes, plus initial state values:

```typescript
export interface Option<T = string> {
  label: string;
  value: T;
}

// Form shape for a page's filter inputs
export interface AnalysisForm {
  product_name: string;
  entity?: string;
  year?: string;
  month?: string;
  quarter?: string;
  threshold?: string;
}

// Payload sent to the API
export interface SubmissionDetails {
  product_name?: string;
  year?: string;
  month?: string;
  threshold?: string;
}

export const initialFormDataValues: SubmissionDetails = {
  product_name: '',
  year: '',
  month: '',
  threshold: '',
};
```

**Pattern:** keep initial state values (`initialFormDataValues`) co-located with the interface in the same file — no need to import default values separately from their type.

## Build and Deploy

```bash
# First: generate the .env from SSM (run from repo root)
python3 combined_terraform/env_generate.py \
    --project-name retail-analytics-dev \
    --region us-east-1

# Build
cd UI
npm install
npm run build        # tsc -b && vite build → outputs to UI/dist/

# Deploy (run from repo root after build)
UI_BUCKET=$(aws ssm get-parameter --name "/retail-analytics-dev/ui/s3-bucket-name" --query Parameter.Value --output text)
CLOUDFRONT_ID=$(aws ssm get-parameter --name "/retail-analytics-dev/ui/cloudfront-distribution-id" --query Parameter.Value --output text)

aws s3 sync UI/dist "s3://${UI_BUCKET}/" --delete
aws cloudfront create-invalidation --distribution-id "${CLOUDFRONT_ID}" --paths "/*"
```

**Local dev:**
```bash
cd UI
cp .env .env.local      # Vite uses .env.local for overrides
npm run dev             # Starts at http://localhost:5173
```

## Adding a New Page

1. Create `src/pages/{PageName}/index.tsx`.
2. Add a route in `src/routes/index.tsx`: `<Route path="/page-name" element={<PageName />} />`.
3. Add a nav item in the layout sidebar component.
4. If the page calls a new API endpoint, add the `VITE_API_*` variable to `env_generate.py` and rebuild the `.env`.
